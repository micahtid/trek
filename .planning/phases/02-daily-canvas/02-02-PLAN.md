**1. Create lib/features/today/presentation/entry_card.dart** — Reusable card widget:

`EntryCard` is a `StatelessWidget` accepting an `Entry` and an `onTap` callback.

Card layout (per user decisions):
- Material 3 Card (elevated or filled, discretion) with rounded corners
- Body text: 2-3 line preview using `maxLines: 3, overflow: TextOverflow.ellipsis`
- Bottom row: creation timestamp formatted with `intl` DateFormat (e.g., "2:30 PM" for today, "Mar 3, 2:30 PM" for other days) on the left
- If `entry.isVoice`: show a subtle mic icon (Icons.mic, small size, muted color) next to the timestamp — this is the voice badge per user decision
- Tap triggers the `onTap` callback (for navigating to detail view)
- Keep spacing, typography, and shadow styling reasonable — this is Claude's discretion

**2. Rewrite lib/features/today/today_screen.dart** — Replace placeholder with card-stream canvas:

`TodayScreen` is a `ConsumerWidget`.

Structure:
- `Scaffold` with:
  - `AppBar`: title "Today" (or just the date), with a search `IconButton` (Icons.search) in actions. The search icon is wired in Plan 03 — for now, just put a placeholder `onPressed` that does nothing or shows a TODO snackbar.
  - `body`: Watch `todayEntriesProvider`:
    - Loading state: `CircularProgressIndicator` centered
    - Error state: error message with retry
    - Empty state (data but zero entries): motivating prompt — centered column with an illustration icon (e.g., `Icons.edit_note` or `Icons.lightbulb_outline`, large, muted color) and text "What did you work on today?" (per user decision)
    - Data state: `ListView.builder` displaying `EntryCard` widgets in reverse chronological order (newest first). Add bottom padding to account for the floating nav bar (~100px so cards don't hide behind it, since AppShell uses `extendBody: true`).
  - `floatingActionButton`: Material 3 FAB with `Icons.add` that calls `_openComposeSheet(context)` (per user decision: FAB opens bottom sheet)

`_openComposeSheet` calls `showModalBottomSheet` with `isScrollControlled: true` (critical for keyboard handling), wraps `ComposeSheet` in `Padding(padding: EdgeInsets.only(bottom: MediaQuery.of(context).viewInsets.bottom))`.

When an `EntryCard` is tapped, navigate to the entry detail screen. Use `Navigator.push` with a `MaterialPageRoute` to `EntryDetailScreen(entry: entry)` — this is a simple push, not a GoRouter route, since entry detail is a transient view, not a tab destination.

**3. Create lib/features/today/presentation/compose_sheet.dart** — Bottom sheet with text + voice:

`ComposeSheet` is a `ConsumerStatefulWidget` (needs StatefulWidget for TextEditingController and speech_to_text state).

Layout:
- `Column` with:
  - Handle bar at top (small centered rounded rectangle, standard bottom sheet indicator)
  - `TextField` with `TextEditingController`, multiline, hint text "What did you work on?", `autofocus: true`
  - Row at bottom:
    - Mic button (`IconButton` with `Icons.mic` / `Icons.mic_off` based on `_isListening` state)
    - Spacer
    - Save button (`FilledButton` or `IconButton` with `Icons.send`) — enabled only when text is non-empty

Voice input (per user decisions — mic button inside compose, unified surface):
- Use `speech_to_text` package. Create `final _speech = stt.SpeechToText();`
- `_toggleListening()`:
  - If already listening: `_speech.stop()`, set `_isListening = false`
  - If not listening: `_speech.initialize()`, then `_speech.listen(onResult: ...)`. On result, set `_controller.text = result.recognizedWords` and move cursor to end. Set `_isListening = true`.
  - Show visual feedback when listening — the mic icon should change (filled vs outlined, or color change, or a pulsing animation — Claude's discretion on the exact feedback per user decision)
- Track `_inputMethod`: starts as `"text"`. When voice is used, set to `"voice"`. If user edits the text afterward, keep it as "voice" (the original input was voice).

Save action:
- Read `currentUserIdProvider` for userId
- Call `ref.read(entryRepositoryProvider).createEntry(userId: userId, body: _controller.text.trim(), inputMethod: _inputMethod)`
- Close the bottom sheet via `Navigator.pop(context)`
- Do NOT save if text is empty (trim and check)

**4. Handle permissions** — `speech_to_text` handles microphone permission requests internally via `_speech.initialize()`. No manual permission code needed. If permission is denied, `initialize()` returns false — show a brief SnackBar informing the user.

**Android permissions:** The executor should verify that `RECORD_AUDIO` permission is declared in `AndroidManifest.xml`. If not already present, add `<uses-permission android:name="android.permission.RECORD_AUDIO"/>` to the manifest.

**1. Create lib/features/today/presentation/entry_detail_screen.dart**:

`EntryDetailScreen` is a `ConsumerStatefulWidget` that receives an `Entry` via constructor.

Layout (per user decisions — tap card to open full view with edit and delete inside):
- `Scaffold` with:
  - `AppBar`: back button (auto from Navigator), title could be the date/time of the entry, optional edit icon in actions
  - `body`: Full entry text display. Initially in read mode.
  - Edit mode: Toggle into edit mode (via an edit button in AppBar or a FAB). In edit mode, show a `TextField` pre-populated with `entry.body`, with a save button.
  - Delete: An `IconButton` (trash icon) in the AppBar or a "Delete" text button. Triggers the undo-delete flow.

**2. Edit flow:**
- Toggle between read mode and edit mode using local state (`_isEditing`)
- In edit mode: `TextField` with `TextEditingController(text: entry.body)`, multiline, autofocus
- Save edit: call `ref.read(entryRepositoryProvider).updateEntry(entryId: entry.id, body: _controller.text.trim())`
- After saving, pop back to the feed (the subscription will auto-update the card)
- If text is empty after trim, show error — don't allow saving empty entries

**3. Delete flow (per user decision — undo snackbar):**
- When user taps delete:
  1. Call `ref.read(entryRepositoryProvider).deleteEntry(entry.id)` — execute the delete immediately
  2. Pop back to the TodayScreen via `Navigator.pop(context)`
  3. Show a SnackBar on the TodayScreen's ScaffoldMessenger: "Entry deleted" with an "Undo" action
  4. Undo action: re-create the entry by calling `ref.read(entryRepositoryProvider).createEntry(userId: entry.userId, body: entry.body, inputMethod: entry.inputMethod)` — the re-created entry gets a new `_id` and `_creationTime` but this is acceptable per research (the undo window is only a few seconds)
  5. SnackBar duration: 4 seconds

**Important:** To show the SnackBar on the parent screen after popping, use `ScaffoldMessenger.of(context)` BEFORE calling `Navigator.pop`. Capture the messenger reference first, then pop, then show the snackbar. Or use the `rootScaffoldMessengerKey` pattern if available.

**4. Entry metadata display:**
- Show creation date/time formatted with `intl` DateFormat
- Show "Voice entry" indicator if `entry.isVoice` (subtle text or icon)
- Full body text with proper typography (not cramped)
