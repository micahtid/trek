**1. Create lib/features/today/presentation/search_screen.dart** — Full-screen search experience:

`SearchScreen` is a `ConsumerStatefulWidget`.

The research recommends `SearchAnchor` but given the need for date range filtering and custom result grouping, a custom full-screen search page gives more control. Use a dedicated screen pushed via `Navigator.push` from TodayScreen.

Layout:
- `Scaffold` with:
  - `AppBar`:
    - Leading: back button
    - Title area: a `TextField` for the search query, `autofocus: true`, hint text "Search entries...", `onChanged` triggers debounced search
    - Optional: a filter icon button that opens date range picker
  - `body`:
    - If query is empty: show helper text ("Search across all your entries") or recent searches placeholder
    - If searching: show `CircularProgressIndicator`
    - If results exist: `ListView` displaying results as `EntryCard` widgets, grouped by date. Group entries by `createdAt` date (year-month-day). Show date headers (e.g., "March 3, 2026", "March 2, 2026") between groups. Each EntryCard is tappable — navigates to `EntryDetailScreen(entry: entry)`.
    - If no results: empty state with icon and "No entries found for '{query}'" text

**Debounced search:**
- Use a `Timer` for debounce (300ms delay after typing stops)
- On each debounced trigger: call `ref.read(entryRepositoryProvider).searchEntries(userId: userId, searchText: query, startDate: startDate, endDate: endDate)`
- Store results in local state (`List<Entry>? _results`, `bool _isSearching`)

**Date range filtering (per user decision — supports full-text search plus optional date range filtering):**
- Add a filter chip or icon button that opens `showDateRangePicker()`
- When a date range is selected, convert start and end dates to millisecondsSinceEpoch
- Pass `startDate` and `endDate` to `searchEntries()`
- Show active filter indicator (e.g., chip showing "Mar 1 - Mar 3" below the search bar)
- Allow clearing the date filter

**Result grouping by date:**
- After getting search results, group entries by date using `entry.createdAt` (year, month, day)
- Display date headers in the ListView using a flat list with mixed types (header items + entry items) or using `ListView.builder` with index-based logic
- Format dates with `intl` DateFormat (e.g., "Monday, March 3, 2026" or "Mar 3, 2026")

**2. Wire search icon in TodayScreen (lib/features/today/today_screen.dart):**

Update the search `IconButton` in TodayScreen's AppBar actions to navigate to SearchScreen:
```dart
IconButton(
  icon: const Icon(Icons.search),
  onPressed: () => Navigator.push(
    context,
    MaterialPageRoute(builder: (_) => const SearchScreen()),
  ),
)
```

This replaces the placeholder onPressed from Plan 02.

**3. Edge cases:**
- Handle case where userId is null (not authenticated) — should not happen since TodayScreen is behind auth guard, but guard defensively
- Handle search text that is only whitespace — treat as empty query
- Handle Convex search errors — show error state with retry option
