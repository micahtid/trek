2. **Create `lib/features/auth/domain/auth_state.dart`** — sealed auth state:
   ```dart
   sealed class AuthState {
     const AuthState();
   }

   class AuthStateUnauthenticated extends AuthState {
     const AuthStateUnauthenticated();
   }

   class AuthStateLoading extends AuthState {
     const AuthStateLoading();
   }

   class AuthStateAuthenticated extends AuthState {
     final String userId;      // Convex user ID
     final String email;
     final String displayName;
     final String? avatarUrl;

     const AuthStateAuthenticated({
       required this.userId,
       required this.email,
       required this.displayName,
       this.avatarUrl,
     });
   }
   ```
   This is the contract — all auth consumers depend on this sealed class, not on Google or Convex internals.

3. **Create `lib/features/auth/data/auth_repository.dart`** — google_sign_in v7 wrapper:

   Use the v7 event stream API (NOT the deprecated v6 signIn/signInSilently).

   Key methods:
   - `initialize()` — calls `GoogleSignIn.instance.initialize()` with `serverClientId` (Web Client ID). Subscribes to `authenticationEvents` stream.
   - `signIn()` — calls `GoogleSignIn.instance.authenticate()`. The result comes via the event stream, not a return value.
   - `signOut()` — calls `GoogleSignIn.instance.signOut()`.
   - `attemptSilentSignIn()` — calls `GoogleSignIn.instance.attemptLightweightAuthentication()`. Used for session restore on app restart.
   - Exposes a `Stream<AuthEvent>` that maps Google's auth events to the app's domain model.

   **CRITICAL per research:**
   - `serverClientId` MUST be the Web OAuth 2.0 Client ID (not iOS/Android). Without this, `idToken` is null on Android (Pitfall 1).
   - Do NOT include any Calendar scopes in `initialize()` — Calendar scope is requested separately in Plan 04 (Pitfall 5).
   - `user.authentication.idToken` is synchronous in v7 (no await needed).
   - `initialize()` must complete before `authenticate()` or `attemptLightweightAuthentication()` is called (Pitfall 4).

   Store the Web Client ID as a constant. The executor should create a config file or constant that can be easily updated by the user.

4. **Create `lib/core/convex/convex_service.dart`** — Convex client with auth bridge:

   Initialize `ConvexClient` with the deployment URL. Wire `setAuthWithRefresh`:
   - `fetchToken` callback: Calls `attemptLightweightAuthentication()` via auth repository, retrieves a fresh Google ID token, returns it as a String (or null if user is signed out).
   - `onAuthChange` callback: Notifies the auth provider when Convex's auth state changes.

   **CRITICAL per research:**
   - Convex calls `fetchToken` on startup AND 60 seconds before token expiry. This is the silent refresh mechanism (Pitfall 3).
   - If `fetchToken` returns null, Convex treats the user as signed out.
   - The `ConvexClient` deployment URL should be configurable (environment or constant).

5. **Create `lib/features/auth/presentation/auth_provider.dart`** — Riverpod notifier:

   An `AsyncNotifier` (or `Notifier`) that:
   - On build: Initializes auth repository, attempts silent sign-in
   - `signIn()`: Triggers Google sign-in flow
   - `signOut()`: Signs out of Google and clears Convex auth
   - Exposes current `AuthState` (unauthenticated, loading, authenticated)
   - Listens to auth repository's event stream and updates state

   The provider must be watchable by go_router for redirect logic.

6. **Create `convex/users.ts`** — Convex mutation for user upsert:
   ```typescript
   import { mutation } from "./_generated/server";
   import { v } from "convex/values";

   // Called after successful Google sign-in to store/update user record
   export const upsertUser = mutation({
     args: {
       googleId: v.string(),
       email: v.string(),
       name: v.string(),
       avatarUrl: v.optional(v.string()),
     },
     handler: async (ctx, args) => {
       const identity = await ctx.auth.getUserIdentity();
       if (!identity) throw new Error("Not authenticated");

       const existing = await ctx.db
         .query("users")
         .withIndex("by_googleId", (q) => q.eq("googleId", args.googleId))
         .unique();

       if (existing) {
         await ctx.db.patch(existing._id, {
           email: args.email,
           name: args.name,
           avatarUrl: args.avatarUrl,
           lastSignIn: Date.now(),
         });
         return existing._id;
       }

       return await ctx.db.insert("users", {
         googleId: args.googleId,
         email: args.email,
         name: args.name,
         avatarUrl: args.avatarUrl,
         lastSignIn: Date.now(),
       });
     },
   });
   ```

**Do NOT:**
- Use `signIn()` or `signInSilently()` — these are removed in google_sign_in v7.
- Store Google ID tokens manually — let google_sign_in manage token caching.
- Add Calendar or GitHub scopes — those are Plan 04.
- Use `jsonwebtoken` for JWT parsing — Convex handles token verification server-side.

2. **Update `lib/core/router/app_router.dart`** — add auth redirect guard:

   Add a `/sign-in` route OUTSIDE the ShellRoute (sign-in screen has no bottom nav).

   Add `redirect` callback to GoRouter:
   ```dart
   redirect: (context, state) {
     final authState = ref.read(authNotifierProvider);
     final isAuthenticated = authState.valueOrNull is AuthStateAuthenticated;
     final isLoading = authState.isLoading;

     if (isLoading) return null; // Don't redirect while loading
     if (!isAuthenticated && state.matchedLocation != '/sign-in') return '/sign-in';
     if (isAuthenticated && state.matchedLocation == '/sign-in') return '/today';
     return null;
   },
   ```

   Add `refreshListenable` so the router reacts to auth state changes. Use a `ChangeNotifier` wrapper around the Riverpod provider, or use `ref.listen` inside the provider to trigger router refresh.

   **CRITICAL:** The `routerProvider` must change from a simple `Provider` to one that has access to `ref` for watching auth state. This likely means making it a `Provider` that accepts `ref`, or restructuring to use `riverpod_annotation` codegen. Choose the simplest approach that works — the key requirement is that the router redirects when auth state changes.
