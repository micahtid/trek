## Decisions Made

**1. applicationID: "verified" as placeholder**
Convex's custom OIDC config requires the `applicationID` to match the `aud` claim in Google ID tokens (which is the Web OAuth 2.0 Client ID). Using `"verified"` as a placeholder causes Convex to accept any Google-signed token during development. This is intentional and documented with a TODO comment. The user must replace this with the actual Web Client ID before production deployment. The actual value comes from Google Cloud Console > APIs & Services > Credentials > Web application OAuth client.

**2. Committed `convex/_generated/` directory**
The Convex CLI generates TypeScript type definitions in `_generated/`. These are committed because: (1) the Flutter/Dart SDK references these types via the convex_flutter package, and (2) they need to be in version control so other developers and CI can use them without running `npx convex dev`.

**3. Excluded `convex/*.js` from git**
The Convex build process transpiles `.ts` source files to `.js` before pushing to the cloud. These are build artifacts regenerated on every `npx convex dev` run and do not belong in version control.

**4. Used `npx convex dev --configure=new` instead of `npx convex init`**
The Convex CLI deprecated the `init` command in favor of `npx convex dev --once --configure=new`. Applied deviation Rule 3 (auto-fix blocking issue) to adapt.

## User Setup Required

**Before Flutter auth wiring (Plan 03) will work end-to-end, the following manual steps are required:**

1. **Create Google Cloud project and OAuth credentials:**
   - Go to https://console.cloud.google.com
   - Create a project (or use existing)
   - Enable the Google Sign-In API
   - Go to APIs & Services > Credentials > Create Credentials > OAuth 2.0 Client ID
   - Create a **Web application** client ID (required for JWT `aud` claim matching)
   - Create an **iOS** client ID (required for iOS builds of the Flutter app)
   - Optionally create an **Android** client ID (Web client ID covers the `serverClientId` need)

2. **Update `convex/auth.config.ts`:**
   - Replace `applicationID: "verified"` with the actual Web Client ID
   - Format: `NUMBERS-HASH.apps.googleusercontent.com`

3. **Update Flutter auth initialization (Plan 03):**
   - Pass the Web Client ID as `serverClientId` in `GoogleSignIn.instance.initialize()`
   - This same Web Client ID must match `applicationID` in auth.config.ts

4. **Redeploy after updating auth.config.ts:**
   ```bash
   npx convex dev --once
   ```
