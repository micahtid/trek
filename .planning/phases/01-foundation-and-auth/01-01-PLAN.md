  <action>
Initialize the Convex backend for this Flutter project. This is a standalone Node/TypeScript backend, not part of the Flutter project tree. Run from the project root:

1. **Install Convex CLI and initialize:**
   ```bash
   npx convex init
   ```
   This creates the `convex/` directory with generated files. If prompted, create a new Convex project named "intern-growth-vault".

2. **Create `convex/auth.config.ts`** with custom OIDC for Google:
   ```typescript
   import { AuthConfig } from "convex/server";

   export default {
     providers: [
       {
         domain: "https://accounts.google.com",
         applicationID: "verified",
         // applicationID MUST be the Web OAuth 2.0 Client ID from Google Cloud Console.
         // The "verified" placeholder means Convex will accept any Google ID token
         // with a valid signature from Google's JWKS endpoint.
         // Replace with actual Web Client ID before production deployment.
         // See: https://docs.convex.dev/auth/advanced/custom-auth
       },
     ],
   } satisfies AuthConfig;
   ```

   **IMPORTANT per research:** The `applicationID` must match the `aud` claim in the Google ID token, which is the **Web OAuth 2.0 Client ID** — NOT the iOS or Android client ID. For initial setup, use the placeholder value. The executor should note in the SUMMARY that the actual Web Client ID must be set by the user after creating it in Google Cloud Console.

3. **Create `convex/schema.ts`** with the users table:
   ```typescript
   import { defineSchema, defineTable } from "convex/server";
   import { v } from "convex/values";

   export default defineSchema({
     users: defineTable({
       // Identity fields from Google ID token (populated by auth)
       googleId: v.string(),       // Google's unique user ID (sub claim)
       email: v.string(),          // User's email address
       name: v.string(),           // Display name
       avatarUrl: v.optional(v.string()), // Profile picture URL

       // Integration tokens (populated in Phase 1)
       githubConnected: v.optional(v.boolean()),

       // Timestamps
       lastSignIn: v.number(),     // Unix timestamp of last sign-in
     })
     .index("by_googleId", ["googleId"])
     .index("by_email", ["email"]),
   });
   ```

4. **Deploy to Convex** to verify the config is accepted:
   ```bash
   npx convex dev --once
   ```
   This pushes the schema and auth config. If auth.config.ts has errors, they will surface here.

**Do NOT:**
- Install Firebase Auth or any other auth library — Convex custom OIDC is the only auth path for Flutter (per STATE.md and research).
- Add a `users` mutation yet — that comes in Plan 03 when auth wiring is built.
- Use `convex auth` command — that's for Convex Auth (React only), not custom OIDC.
