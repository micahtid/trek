<objective>
Build the AI drilldown chat flow: after saving an entry, user is auto-navigated to entry detail where Gemini asks 3-4 follow-up questions in a mini chat. When complete, the AI merges answers into the entry body as a polished paragraph, then runs the analysis pipeline (skill tagging, impact, categories) and persists results to Convex.

Purpose: This is the core differentiator -- the AI that "won't accept vague entries." It transforms raw reflections into specific, quantified, resume-ready content through a conversational coaching interaction.

Output: Working drilldown chat with streaming Gemini responses, merge pipeline, analysis pipeline, and full persistence of AI metadata.
</objective>

Create `lib/features/ai/presentation/ai_providers.dart` with:

**1. `DrilldownNotifier` (StateNotifier<DrilldownState>):**
- Takes `entryId` (String), `originalBody` (String), `Ref` in constructor.
- On construction: sets status to `active`, creates a `ChatSession` via `AiService.instance.startDrilldown(originalBody)`, stores it as a private field. Immediately sends the entry body as the first user message to get the AI's first question (call `_requestAiQuestion()`).
- Also on construction: updates entry's `aiDrilldownStatus` to `'in_progress'` via `EntryRepository.updateAiMetadata()`.
- `_requestAiQuestion()`: Sets status to `waitingForAi`. Calls `AiService.instance.sendMessageStream(session, ...)` and collects the streamed text. When stream completes, adds an AI ChatMessage to state, increments roundCount. Sets status back to `active`. If roundCount >= 4, auto-completes. Exposes a `streamingText` field (not in DrilldownState, but as a separate public getter on the notifier) that updates during streaming so the UI can show typewriter effect.
- `addUserMessage(String text)`: Adds a user ChatMessage to state messages, then calls `_requestAiQuestion()`.
- `finishDrilldown()`: Called when user taps "I'm done" or roundCount hits max. Sets status to `waitingForAi` (show loading). Runs the completion pipeline:
  1. Extract user answers from messages (messages where isUser == true, excluding the original entry).
  2. Call `AiService.instance.mergeEntryBody(originalBody, answers)` to get merged text.
  3. Call `AiService.instance.analyzeEntry(mergedText)` to get AiAnalysisResult.
  4. Call `EntryRepository.updateAiMetadata()` with: body=mergedText, skillTags, impactLevel, primaryCategory, secondaryCategories, aiDrilldownStatus='complete', aiDrilldownMessages (serialized messages).
  5. Set state to complete with enrichedBody = mergedText.
  6. Invalidate `todayEntriesProvider` so the feed refreshes with new body/metadata.
- `skipDrilldown()`: Sets aiDrilldownStatus to 'skipped' in Convex, sets state status to `skipped`.
- Handle errors: wrap AI calls in try-catch, set status to `error` with errorMessage on failure. Include a `retry()` method that resets error state and retries the last operation.
- Persist drilldown messages to Convex after each exchange (user message + AI response) so the conversation is resumable. Call `updateAiMetadata(entryId: ..., aiDrilldownMessages: messages.map((m) => m.toJson()).toList())`.

**2. `drilldownProvider`:**
- `StateNotifierProvider.family.autoDispose<DrilldownNotifier, DrilldownState, ({String entryId, String originalBody})>`
- Creates DrilldownNotifier with the entry ID and original body.

**3. `resumeDrilldownProvider`:**

If so, returns the messages as List<ChatMessage> so the UI can display the "Continue" option with prior context.
